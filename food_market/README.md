## 🎯 Описание проекта

Данная программная система реализует модель **продуктового рынка** с интерфейсом командной строки (CLI). Система позволяет моделировать взаимодействие между покупателями, продавцами и товарами на рынке, включая такие операции, как выбор продуктов, торг, применение акций, оплата покупок и контроль сроков годности.

Проект разработан в рамках лабораторной работы по курсу "Проектирование программного обеспечения интеллектуальных систем".

## 🏪 Предметная область

**Продуктовый рынок** — это место, где происходит купля-продажа продовольственных товаров. В системе моделируются следующие аспекты:

- Покупатели выбирают продукты из доступного ассортимента
- Продавцы могут торговаться с покупателями
- Действуют акции и скидки на определенные категории товаров
- Важен контроль сроков годности продуктов
- Производится расчет и оплата покупок

### Важные сущности

| Сущность | Описание |
|----------|----------|
| **Рынок (Market)** | Основное место торговли, содержит продавцов, товары и акции |
| **Покупатель (Buyer)** | Пользователь системы, выбирает товары и оплачивает покупки |
| **Продавец (Seller)** | Владелец товара, может торговаться |
| **Продукт (Product)** | Товар с названием, ценой, сроком годности и количеством |
| **Акция (Promotion)** | Скидочное предложение на определенные товары |
| **Корзина (Cart)** | Временное хранилище выбранных товаров |

## ⚙️ Функциональные возможности

### Основные операции (обязательные):

| № | Операция | Описание |
|---|----------|----------|
| 1 | 🔍 **Выбор продуктов** | Покупатель выбирает товары из каталога и добавляет в корзину |
| 2 | 💰 **Торговля с продавцом** | Покупатель может предложить свою цену на товар |
| 3 | 🧾 **Расчет за покупки** | Подсчет итоговой суммы с учетом акций |
| 4 | 🎁 **Участие в акциях** | Активация скидочных предложений |
| 5 | ⚠️ **Проверка сроков годности** | Выявление просроченных товаров |

### Дополнительные возможности:

- Просмотр всех доступных товаров с информацией о годности
- Отображение содержимого корзины
- Оплата покупок с проверкой достаточности средств
- Обработка ошибочных ситуаций (недостаточно средств, товар просрочен и т.д.)

## 🏗️ Архитектура проекта

Проект построен на принципах **объектно-ориентированного анализа** и разделения ответственности.

### Модули:

| Файл | Ответственность |
|------|----------------|
| `models.py` | Определение классов: `Product`, `Seller`, `Buyer`, `Promotion`, `Market` |
| `services.py` | Реализация операций: выбор товаров, торг, расчет, оплата, проверка годности |
| `cli.py` | Пользовательский интерфейс, меню, взаимодействие с пользователем |
| `tests.py` | Набор unit-тестов для проверки корректности работы |
| `main.py` | Инициализация и запуск приложения |

### Обработка ошибок:

В системе используются пользовательские исключения:
- `ProductExpiredError` — при попытке купить просроченный товар
- `InsufficientFundsError` — при недостатке средств у покупателя
- `ProductNotFoundError` — при попытке выбрать несуществующий товар

## 🚀 Запуск программы

### 1. Убедитесь, что установлен Python 3.10 или выше
python --version
### 2. Скачайте проект
git clone https://github.com/GlebMitko/food_market.git
cd food_market
### 3. Запустите приложение
python main.py
или (macOS/Linux):
python3 main.py
### 4. Пример работы

После запуска появится меню:

5. Показать корзину и итоговую сумму
