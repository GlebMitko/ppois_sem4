## 🏪 Общее описание системы

**Предметная область:** Продуктовый рынок (вариант №9)

Система моделирует работу продуктового рынка:

- Покупатели могут просматривать товары
- Добавлять товары в корзину
- Торговаться с продавцами
- Активировать акции и скидки
- Осуществлять оплату покупок
- Контролировать сроки годности продуктов

### Основные сущности

| Сущность | Описание |
|----------|----------|
| **Market** | Рынок, объединяет продавцов, товары и акции |
| **Buyer** | Покупатель с именем и балансом счета |
| **Seller** | Продавец с именем и номером ларька |
| **Product** | Товар (название, цена, срок годности, количество) |
| **Promotion** | Акция (скидка на определенные товары) |

### Операции (из ЛР №1)

1. Выбор продуктов в корзину
2. Торговля с продавцом
3. Расчет за покупки с учетом акций
4. Участие в акциях
5. Проверка сроков годности

## 🏗️ Архитектура MVC

### Что такое MVC?

**MVC (Model-View-Controller)** — это архитектурный паттерн, разделяющий приложение на три взаимосвязанных компонента.

### Active Model vs Passive Model

| Тип | Описание | В нашем проекте |
|-----|----------|-----------------|
| **Active Model** | Модель сама уведомляет View об изменениях | Частично используется (Controller обновляет View) |
| **Passive Model** | Модель не знает о View, Controller управляет обновлением | **Применяется** — Model не зависит от View |

### Отличие Controller в MVC от Presenter в MVP

| Паттерн | Роль | Взаимодействие с View |
|---------|------|----------------------|
| **MVC** | Controller обрабатывает ввод и обновляет Model | View передает события Controller |
| **MVP** | Presenter управляет View и Model напрямую | View пассивна, Presenter обновляет ее |

В нашем проекте используется **классический MVC**.


### Назначение модулей

| Файл | Ответственность |
|------|-----------------|
| `model/market_model.py` | Domain-модель (сущности предметной области) |
| `model/market_service.py` | Бизнес-операции (выбор, оплата, акции, исключения) |
| `cli/cli_app.py` | Консольный интерфейс (меню 1-7, ввод с клавиатуры) |
| `gui/mvc_views.py` | Графическое представление (окна, таблицы, кнопки) |
| `gui/mvc_controllers.py` | Обработчики событий GUI (клики, ввод текста) |
| `gui/gui_app.py` | Инициализация PyQt5 и запуск приложения |
| `main.py` | Выбор режима (CLI или GUI) |
| `tests.py` | Unit-тесты для бизнес-логики |
